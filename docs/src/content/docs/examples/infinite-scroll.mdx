# Infinite Scroll

IntersectionTrigger can power infinite scroll patterns using a **sentinel element** -- an invisible element placed at the bottom of your content list. When the sentinel enters the viewport, you load more content and optionally move or recreate the sentinel.

## Basic Infinite Scroll

```js
import IntersectionTrigger from 'intersectiontrigger';

const it = new IntersectionTrigger({
	rootEnter: '100%',
});

it.add('#sentinel', {
	onEnter() {
		loadMoreContent();
	},
});

function loadMoreContent() {
	// Fetch or generate new content
	const newItems = fetchNextPage();

	const container = document.getElementById('content-list');
	newItems.forEach((item) => {
		const el = document.createElement('div');
		el.className = 'item';
		el.textContent = item.title;
		container.appendChild(el);
	});
}
```

## HTML

```html
<div id="content-list">
	<div class="item">Item 1</div>
	<div class="item">Item 2</div>
	<div class="item">Item 3</div>
	<!-- More items... -->
</div>

<div id="sentinel"></div>
```

## CSS

The sentinel is invisible but needs a small presence in the layout:

```css
#sentinel {
	width: 100%;
	height: 1px;
}

.item {
	padding: 16px;

}
```

## The Sentinel Pattern

The sentinel pattern works by placing a marker element at the end of your scrollable content. When this marker becomes visible, it signals that the user has scrolled far enough to need more content.

### Advantages over scroll event listeners

- **No scroll calculation**: You do not need to compute scroll positions or distances to the bottom.
- **Automatic throttling**: IntersectionObserver handles throttling internally.
- **Root-aware**: Works correctly with any scrollable container, not just the viewport.
- **Clean teardown**: Remove the sentinel observation when there is no more content to load.

## Handling Pagination

A real-world implementation tracks the current page and stops loading when all data is fetched:

```js
let currentPage = 1;
let hasMore = true;

const it = new IntersectionTrigger({
	rootEnter: '100%',
});

it.add('#sentinel', {
	onEnter() {
		if (!hasMore) return;
		loadMoreContent();
	},
});

async function loadMoreContent() {
	const response = await fetch(`/api/items?page=${currentPage}`);
	const data = await response.json();

	const container = document.getElementById('content-list');

	data.items.forEach((item) => {
		const el = document.createElement('div');
		el.className = 'item';
		el.textContent = item.title;
		container.appendChild(el);
	});

	currentPage++;
	hasMore = data.hasMore;

	// No more content -- stop observing the sentinel
	if (!hasMore) {
		it.remove('#sentinel');
		document.getElementById('sentinel').remove();
	}
}
```

## Dynamic Sentinels

In some cases you may want to recreate the sentinel after loading new content. This is useful if the DOM structure changes significantly:

```js
function loadMoreContent() {
	const container = document.getElementById('content-list');

	// Remove old sentinel
	const oldSentinel = document.getElementById('sentinel');
	it.remove(oldSentinel);

	// Append new items
	newItems.forEach((item) => {
		const el = document.createElement('div');
		el.className = 'item';
		el.textContent = item.title;
		container.appendChild(el);
	});

	// Create new sentinel at the bottom
	const newSentinel = document.createElement('div');
	newSentinel.id = 'sentinel';
	container.after(newSentinel);

	// Observe the new sentinel
	it.add(newSentinel, {
		onEnter() {
			loadMoreContent();
		},
	});
}
```

## How It Works

<Steps>

1. **Place a sentinel element**
   Add an empty `div` at the bottom of your content area. It does not need to be visible.

2. **Observe the sentinel**
   Use `it.add()` with an `onEnter` callback that loads more content.

3. **Load content on intersection**
   When the user scrolls near the bottom and the sentinel becomes visible, the callback fires. Fetch new data and append it to the content container.

4. **Clean up when done**
   When there is no more content to load, remove the sentinel from the DOM and call `it.remove()` to stop observing it.

</Steps>

## Notes

- Use `rootEnter: '100%'` or a pixel value like `'300px'` to start loading before the user reaches the very bottom. This prevents a visible loading gap.
- Avoid using `once: true` for infinite scroll unless you dynamically recreate the sentinel each time.
- You can show a loading spinner inside the sentinel element while content is being fetched.
- For large lists, consider virtual scrolling in addition to infinite scroll to keep DOM node counts manageable.
