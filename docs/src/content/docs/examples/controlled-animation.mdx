# Controlled Animation

The Animation plugin lets you tie anime.js animations to intersection events. Instead of manually toggling classes, you can play, reverse, or restart fully-defined animations when elements enter or leave the viewport.

## Basic Usage

```js
import IntersectionTrigger, { Animation } from 'intersectiontrigger';
import anime from 'animejs';

IntersectionTrigger.registerPlugins([Animation]);

const it = new IntersectionTrigger();

it.add('.element', {
	animation: anime({
		targets: '.element',
		translateY: [50, 0],
		opacity: [0, 1],
		duration: 800,
		easing: 'easeOutCubic',
		autoplay: false,
	}),
});
```

:::note
Always set `autoplay: false` on your anime.js instance. The Animation plugin takes control of playback based on intersection events.
:::

## Toggle Actions

The `toggleActions` property controls what happens at each intersection event. It accepts a space-separated string of four values in this order:

**enter | leave | enterBack | leaveBack**

Each position can be one of:

| Value      | Behavior                                    |
| ---------- | ------------------------------------------- |
| `play`     | Plays the animation from its current state  |
| `pause`    | Pauses the animation                        |
| `restart`  | Restarts the animation from the beginning   |
| `reverse`  | Reverses the animation                      |
| `complete` | Jumps the animation to its end state        |
| `none`     | Does nothing (watchdog is unaffected)       |

### Default Toggle Actions

The default value is `'play complete reverse complete'`:

- **enter** -- play the animation forward
- **leave** -- jump to the completed state
- **enterBack** -- reverse the animation back
- **leaveBack** -- jump to the completed state

## Custom Toggle Actions

<Tabs>
<TabItem label="Fade in only">

```js
it.add('.fade-in', {
	animation: anime({
		targets: '.fade-in',
		opacity: [0, 1],
		duration: 600,
		easing: 'easeOutQuad',
		autoplay: false,
	}),
	toggleActions: 'play none reverse none',
});
```

</TabItem>
<TabItem label="Restart on re-entry">

```js
it.add('.restart', {
	animation: anime({
		targets: '.restart',
		translateX: [0, 200],
		duration: 1000,
		easing: 'easeInOutSine',
		autoplay: false,
	}),
	toggleActions: 'restart none restart none',
});
```

</TabItem>
<TabItem label="One-shot animation">

```js
it.add('.once', {
	animation: anime({
		targets: '.once',
		scale: [0.5, 1],
		opacity: [0, 1],
		duration: 500,
		easing: 'easeOutBack',
		autoplay: false,
	}),
	toggleActions: 'play none none none',
});
```

## How It Works

<Steps>

1. **Register the plugin**
   Call `IntersectionTrigger.registerPlugins([Animation])` before creating any instances. This adds animation support to the IntersectionTrigger pipeline.

2. **Create an animation**
   Define an anime.js animation with `autoplay: false`. The plugin will control playback.

3. **Pass it to `add()`**
   Use the `animation` option to attach the animation. The plugin reads `toggleActions` (or uses the defaults) to determine what happens at each intersection event.

4. **Scroll to see it**
   As the element enters and leaves the viewport, the animation plays, pauses, reverses, or completes according to the toggle actions you defined.

</Steps>

## Notes

- The `animation` option accepts a single anime.js instance or an object with `instance` and optional `link` / `snap` properties (see [Linked Animation](./linked-animation/) and [Snapping](./snapping/)).
- You can combine `animation` with `onEnter` / `onLeave` callbacks for additional side effects.
- The Animation plugin requires anime.js to be installed as a peer dependency.
