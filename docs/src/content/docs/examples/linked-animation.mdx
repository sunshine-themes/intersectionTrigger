# Linked Animation

Linked animations tie animation progress directly to scroll position. Instead of simply playing or reversing on intersection, the animation scrubbs forward and backward as the user scrolls. This creates smooth, scroll-driven effects.

## Basic Linked Animation

```js
import IntersectionTrigger, { Animation } from 'intersectiontrigger';
import anime from 'animejs';

IntersectionTrigger.registerPlugins([Animation]);

const it = new IntersectionTrigger();

it.add('.progress-bar', {
	animation: {
		instance: anime({
			targets: '.progress-fill',
			width: '100%',
			duration: 1000,
			easing: 'linear',
			autoplay: false,
		}),
		link: 3,
	},
});
```

## Link Factor

The `link` property controls how tightly the animation follows scroll progress. It accepts:

| Value           | Behavior                                               |
| --------------- | ------------------------------------------------------ |
| `true` or `1`   | Animation progress matches scroll progress 1:1         |
| A number > 1    | Animation catches up faster (smoothing is reduced)     |
| `false`         | Disables linking (falls back to controlled animation)  |

A higher link factor means the animation reaches its target progress more quickly after each scroll event. A lower value (closer to 1) creates a smoother, more gradual feel.

### When to use different values

- **`link: true` or `link: 1`** -- Parallax effects, fade-ins where you want precise scroll sync.
- **`link: 3` to `link: 6`** -- Progress bars, counters, and UI elements where some smoothing feels natural.
- **`link: 10+`** -- Near-instant tracking with minimal smoothing, useful when you want responsive feedback.

## Progress Bar Example

### HTML

```html
<div class="progress-bar">
	<div class="progress-fill"></div>
</div>
```

### CSS

```css
.progress-bar {
	width: 100%;
	height: 8px;
	background: #e2e8f0;
	border-radius: 4px;
	overflow: hidden;
}

.progress-fill {
	width: 0%;
	height: 100%;
	background: linear-gradient(90deg, #6366f1, #8b5cf6);
	border-radius: 4px;
}
```

### JavaScript

```js
it.add('.progress-bar', {
	animation: {
		instance: anime({
			targets: '.progress-fill',
			width: '100%',
			duration: 1000,
			easing: 'linear',
			autoplay: false,
		}),
		link: 3,
	},
});
```

As you scroll past the `.progress-bar` element, the fill width animates from 0% to 100% in lockstep with scroll position.

## Linked Animation with Toggle Actions

You can combine `link` with `toggleActions` to control what happens when the animation completes or reverses:

```js
it.add('.parallax-image', {
	animation: {
		instance: anime({
			targets: '.parallax-image',
			translateY: [100, -100],
			duration: 1000,
			easing: 'linear',
			autoplay: false,
		}),
		link: true,
	},
	toggleActions: 'play none reverse none',
});
```

## How It Works

<Steps>

1. **Register the Animation plugin**
   As with controlled animations, call `IntersectionTrigger.registerPlugins([Animation])` first.

2. **Use the object syntax**
   Instead of passing an anime.js instance directly to `animation`, pass an object with `instance` and `link` properties.

3. **Set the link factor**
   Choose a value that suits the desired feel. Start with `link: 3` and adjust up or down.

4. **Scroll to scrub**
   The animation progress is now driven by how far the element has scrolled through the intersection zone. Scrolling back reverses the animation smoothly.

</Steps>

## Notes

- The `easing` in the anime.js instance affects the animation curve within the scroll range. Use `linear` for true 1:1 scroll mapping.
- Linked animations can be combined with [Snapping](./snapping/) to create scroll-snap effects.
- `link` only affects animation playback. `onEnter` and `onLeave` callbacks still fire at their normal intersection thresholds.
