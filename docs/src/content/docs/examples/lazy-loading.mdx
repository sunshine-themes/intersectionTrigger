# Lazy Loading

IntersectionTrigger is a natural fit for lazy loading images. By using the `once: true` option, callbacks fire only the first time an element enters the viewport -- perfect for one-time operations like loading images.

## Basic Lazy Loading

```js
import IntersectionTrigger from 'intersectiontrigger';

const it = new IntersectionTrigger();

it.add('img[data-src]', {
	once: true,
	onEnter(trigger) {
		trigger.src = trigger.dataset.src;
		trigger.classList.add('loaded');
	},
});
```

## HTML

Use the `data-src` pattern to defer the actual image URL until the element is visible:

```html
<img data-src="/images/photo-large.jpg" alt="A scenic view" class="lazy-image" />
<img data-src="/images/portrait.jpg" alt="A portrait photo" class="lazy-image" />
<img data-src="/images/landscape.jpg" alt="A landscape" class="lazy-image" />
```

## CSS

```css
.lazy-image {
	opacity: 0;
	transition: opacity 0.4s ease;
	background: #e2e8f0;
	min-height: 200px;
}

.lazy-image.loaded {
	opacity: 1;
}
```

## Why `once: true`

The `once` option unobserves the element after the first intersection. For lazy loading, this is the desired behavior:

- **Performance**: Once the image is loaded, there is no reason to keep observing it. The observer stops watching, freeing resources.
- **No flicker**: Without `once`, the image would re-trigger `onEnter` every time it scrolls back into view, redundantly reassigning `src`.
- **Simplicity**: No need to manually call `it.remove()` or track which images have been loaded.

## Loading with Placeholder

For a smoother experience, use a low-quality placeholder as the initial `src` and swap in the full image on intersection:

```html
<img
	src="/images/photo-thumb.jpg"
	data-src="/images/photo-large.jpg"
	alt="A scenic view"
	class="lazy-image"
/>
```

```js
it.add('img[data-src]', {
	once: true,
	onEnter(trigger) {
		const fullSrc = trigger.dataset.src;

		// Wait for the full image to load before swapping
		const img = new Image();
		img.src = fullSrc;
		img.onload = () => {
			trigger.src = fullSrc;
			trigger.classList.add('loaded');
		};
	},
});
```

## Lazy Loading Backgrounds

The same pattern works for elements with background images:

```js
it.add('[data-bg]', {
	once: true,
	onEnter(trigger) {
		trigger.style.backgroundImage = `url(${trigger.dataset.bg})`;
		trigger.classList.add('loaded');
	},
});
```

```html
<div data-bg="/images/hero.jpg" class="hero-section">
	<h1>Welcome</h1>
</div>
```

## How It Works

<Steps>

1. **Mark images with `data-src`**
   Instead of setting `src` directly, store the URL in a `data-src` attribute. The browser will not load the image.

2. **Create a trigger with `once: true`**
   This ensures the callback fires exactly once per element.

3. **Swap `data-src` into `src`**
   In the `onEnter` callback, copy the `data-src` value to `src`. The browser begins loading the image immediately.

4. **Add a loaded class**
   Apply a CSS transition for a smooth fade-in as the image loads.

</Steps>

## Notes

- Modern browsers have native lazy loading (`loading="lazy"`), but IntersectionTrigger gives you full control over the trigger threshold, root margin, and callbacks.
- You can adjust when loading begins by setting `rootEnter` on the constructor. For example, `rootEnter: '200px'` starts loading 200px before the image enters the viewport.
- For large image galleries, consider combining with a debounce or limiting concurrent loads.
