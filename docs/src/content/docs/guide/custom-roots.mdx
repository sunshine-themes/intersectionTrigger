## What Is a Root Element?

The **root** is the scrollable container that defines the intersection boundary. IntersectionTrigger monitors trigger elements relative to this root. When a trigger crosses the root's configured boundaries, callbacks fire.

By default, the root is `null`, which means the **viewport** (the browser window) is used -- equivalent to how the native `IntersectionObserver` behaves with a `null` root.

## Setting a Custom Root

Pass the `root` option to the constructor. You can provide either a **CSS selector string** or an **HTMLElement reference**.

### Using a CSS Selector String

```js
import IntersectionTrigger from 'intersectiontrigger';

const it = new IntersectionTrigger({
	root: '#scroll-container',
});

it.add('.card', {
	onEnter: (trigger) => trigger.classList.add('visible'),
	onLeave: (trigger) => trigger.classList.remove('visible'),
});
```

### Using an Element Reference

```js
const container = document.querySelector('#scroll-container');

const it = new IntersectionTrigger({
	root: container,
});
```

The root element must be a scrollable ancestor of the trigger elements. If the element does not actually overflow and scroll, no intersection changes will be detected.

### Using the Viewport

To explicitly use the viewport (or to accept the default), pass `null` or omit the `root` option entirely:

```js
// These are equivalent
const it = new IntersectionTrigger();
const it2 = new IntersectionTrigger({ root: null });
```

## rootEnter and rootLeave Positioning

`rootEnter` and `rootLeave` define **where on the root boundary** the intersection is evaluated. They are measured from the **top edge** of the root (for the Y-axis) or the **left edge** (for the X-axis).

### Value Format

Both accept:

- **Percentage strings** relative to the root dimension: `'0%'`, `'50%'`, `'100%'`
- **Pixel strings** as absolute offsets: `'200px'`, `'50px'`
- **Functions** that return a value string, receiving the instance as an argument:

```ts
rootEnter: (it) => `${it.rootBounds.height * 0.75}px`
```

### Default Values

| Option | Default | Meaning |
|--------|---------|---------|
| `rootEnter` | `'100%'` | The bottom edge of the root (Y-axis) or right edge (X-axis). |
| `rootLeave` | `'0%'` | The top edge of the root (Y-axis) or left edge (X-axis). |

With the defaults, a trigger entering from below the root is detected at the bottom of the root (`100%`), and the leave event is detected at the top of the root (`0%`).

### Adjusting Detection Points

Move the detection points inward for an earlier trigger, or outward for a later one:

```js
const it = new IntersectionTrigger({
	root: '#scroll-container',
	// Detect enter when trigger reaches 75% down the root
	rootEnter: '75%',
	// Detect leave when trigger passes 25% from the top of the root
	rootLeave: '25%',
});
```

Or use pixel values for precise, layout-specific positioning:

```js
const it = new IntersectionTrigger({
	root: '#scroll-container',
	// Detect enter 200px from the top of the root
	rootEnter: '200px',
	// Detect leave 100px from the top of the root
	rootLeave: '100px',
});
```

### Dynamic rootEnter / rootLeave

Pass a function to compute the position dynamically at construction time:

```js
const it = new IntersectionTrigger({
	root: '#scroll-container',
	rootEnter: (it) => {
		// Enter detection at 80% of root height
		return `${(it.rootBounds.height / 100) * 80}%`;
	},
	rootLeave: '0%',
});
```

## Full Example

A scrollable panel with cards that animate in as they cross custom root boundaries:

```js
import IntersectionTrigger from 'intersectiontrigger';

const it = new IntersectionTrigger({
	root: '.panel',
	rootEnter: '85%',
	rootLeave: '15%',
	defaults: {
		enter: '0%',
		leave: '100%',
		onEnter: (trigger) => {
			trigger.style.opacity = '1';
			trigger.style.transform = 'translateY(0)';
		},
		onLeave: (trigger) => {
			trigger.style.opacity = '0';
			trigger.style.transform = 'translateY(20px)';
		},
		onEnterBack: (trigger) => {
			trigger.style.opacity = '1';
			trigger.style.transform = 'translateY(0)';
		},
		onLeaveBack: (trigger) => {
			trigger.style.opacity = '0';
			trigger.style.transform = 'translateY(-20px)';
		},
	},
});

it.add('.card');
```

## HTML Structure Requirement

For a custom root to work, the root element must be a **scrollable ancestor** of the trigger elements. A typical HTML structure looks like this:

```html
<div id="scroll-container" style="height: 400px; overflow-y: auto;">
	<div class="card">Card 1</div>
	<div class="card">Card 2</div>
	<div class="card">Card 3</div>
	<!-- more cards... -->
</div>
```

The `#scroll-container` element must have a constrained height (or width for horizontal scrolling) and `overflow: auto` or `overflow: scroll` so that its content overflows and becomes scrollable.
