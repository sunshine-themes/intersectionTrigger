## Instance Methods

### `add(trigger, options?)`

Adds one or more elements as triggers to observe. Returns the IntersectionTrigger instance for chaining.

```ts
add(trigger: string | HTMLElement | HTMLElement[] | NodeListOf<HTMLElement>, options?: TriggerOptions): this
```

The `trigger` argument accepts a CSS selector string, a single element, an array of elements, or a `NodeListOf<HTMLElement>`.

#### Trigger Options

These options can be passed as the second argument to `add()`, or set as `defaults` in the constructor to apply to all triggers.

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `enter` | `TriggerPosition` | `'0%'` | The position on the trigger element that must meet `rootEnter` to fire the enter callback. Accepts a percentage string or a function returning one. |
| `leave` | `TriggerPosition` | `'100%'` | The position on the trigger element that must meet `rootLeave` to fire the leave callback. Same value format as `enter`. |
| `once` | `boolean` | `false` | If `true`, the trigger is automatically removed after its first leave event. |
| `onEnter` | [`ItCallbackFunction`](/guide/callbacks) | `undefined` | Callback fired when the trigger enters in the forward (positive) direction. |
| `onLeave` | [`ItCallbackFunction`](/guide/callbacks) | `undefined` | Callback fired when the trigger leaves in the forward (positive) direction. |
| `onEnterBack` | [`ItCallbackFunction`](/guide/callbacks) | `undefined` | Callback fired when the trigger enters in the backward (negative) direction. |
| `onLeaveBack` | [`ItCallbackFunction`](/guide/callbacks) | `undefined` | Callback fired when the trigger leaves in the backward (negative) direction. |
| `toggleClass` | `string \| ToggleClassOptions[]` | `undefined` | CSS class toggling configuration. Pass a class name string or an array of detailed toggle configs. |
| `animation` | `AnimeInstance \| AnimationOptions` | `undefined` | Animation configuration. Pass an AnimeJS instance or a detailed animation options object. |

#### Example

```js
const it = new IntersectionTrigger({ root: '#container' });

// Add with a selector
it.add('.card', {
	enter: '20%',
	leave: '80%',
	onEnter: (trigger) => trigger.classList.add('in-view'),
	onLeave: (trigger) => trigger.classList.remove('in-view'),
});

// Add with an element reference
const el = document.querySelector('#hero');
it.add(el, {
	once: true,
	onEnter: (trigger, instance) => {
		console.log('Hero entered', trigger, instance);
	},
});

// Add multiple elements at once
it.add(document.querySelectorAll('.fade-in'));
```

### `remove(trigger)`

Stops observing the specified element(s) and removes them from the instance. Returns the instance for chaining.

```ts
remove(trigger: string | HTMLElement | HTMLElement[] | NodeListOf<HTMLElement>): this
```

```js
const it = new IntersectionTrigger();
it.add('.card');

// Remove by selector
it.remove('.card');

// Remove by element reference
const el = document.querySelector('#hero');
it.remove(el);
```

---

### `update()`

Disconnects and recreates the internal `IntersectionObserver`, recalculates all positions, and re-observes all triggers. Call this after layout changes that affect trigger or root dimensions (e.g., window resize is handled automatically).

```ts
update(): void
```

```js
const it = new IntersectionTrigger({ root: '#container' });
it.add('.card');

// After a dynamic layout change
it.update();
```

---

### `kill()`

Destroys the instance completely. Disconnects the observer, removes all event listeners, kills any active plugins, and removes the instance from the global registry.

```ts
kill(): void
```

```js
const it = new IntersectionTrigger();
it.add('.card');

// Clean up
it.kill();
```

## Static Methods

### `IntersectionTrigger.getInstances()`

Returns an array of all active IntersectionTrigger instances.

```ts
static getInstances(): IntersectionTrigger[]
```

```js
const all = IntersectionTrigger.getInstances();
console.log(`${all.length} active instances`);
```

---

### `IntersectionTrigger.getInstanceById(id)`

Returns the instance with the given numeric ID, or `undefined` if not found.

```ts
static getInstanceById(id: number): IntersectionTrigger | undefined
```

```js
const it = new IntersectionTrigger();
const same = IntersectionTrigger.getInstanceById(it.id);
console.log(same === it); // true
```

---

### `IntersectionTrigger.registerPlugins(plugins)`

Registers plugins to be available to all instances. Returns the new count of registered plugins.

```ts
static registerPlugins(plugins: Plugin[]): number
```

```js
import IntersectionTrigger from 'intersectiontrigger';
import Animation from 'intersectiontrigger/plugins/animation';
import ToggleClass from 'intersectiontrigger/plugins/toggle-class';
import Guides from 'intersectiontrigger/plugins/guides';

IntersectionTrigger.registerPlugins([Animation, ToggleClass, Guides]);
```

---

### `IntersectionTrigger.getRegisteredPlugins()`

Returns the array of currently registered plugin constructors.

```ts
static getRegisteredPlugins(): Plugin[]
```

```js
const plugins = IntersectionTrigger.getRegisteredPlugins();
console.log(plugins); // [Animation, ToggleClass, Guides]
```

---

### `IntersectionTrigger.update()`

Calls `update()` on every active instance. Useful after a global layout change.

```ts
static update(): void
```

```js
IntersectionTrigger.update();
```

---

### `IntersectionTrigger.kill()`

Kills every active instance, removing all observers, listeners, and state.

```ts
static kill(): void
```

```js
IntersectionTrigger.kill();
```
