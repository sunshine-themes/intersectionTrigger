---
title: ToggleClass Plugin
description: Toggle CSS classes on intersection events
---

The ToggleClass plugin adds or removes CSS classes on elements when intersection events occur. It supports both a concise string shorthand and a detailed array configuration.

## Registration

```js
import { IntersectionTrigger, ToggleClass } from 'intersectiontrigger';

IntersectionTrigger.registerPlugins([ToggleClass]);
```

## String Shorthand

The simplest way to use ToggleClass is to pass a space-separated string of class names. These classes are added on enter and removed on leave, using the default toggle actions (`'add remove add remove'`).

```js
const it = new IntersectionTrigger();

it.add('.box', {
  toggleClass: 'active visible'
});
```

This toggles the `active` and `visible` classes on each `.box` element when it enters or leaves the intersection zone.

## Array Configuration

For more control, pass an array of `ToggleClassOptions` objects. Each object can target different elements with different class names and actions.

### ToggleClassOptions

| Property | Type | Default | Description |
| -------- | ---- | ------- | ----------- |
| `targets` | `Trigger` | trigger element | Element(s) to toggle classes on. Accepts a selector string, HTMLElement, array, or NodeList |
| `classNames` | `string` | *required* | Space-separated class names to toggle |
| `toggleActions` | `string` | `'add remove add remove'` | Space-separated actions for enter/leave/enterBack/leaveBack |

### Available Actions

| Action | Description |
| ------ | ----------- |
| `add` | Adds the class to the element |
| `remove` | Removes the class from the element |
| `none` | No action for this event |

### Multiple Targets Example

```js
it.add('.card', {
  toggleClass: [
    {
      targets: '.card',
      classNames: 'in-view'
    },
    {
      targets: '.card-title',
      classNames: 'highlight',
      toggleActions: 'add none remove none'
    },
    {
      targets: '.card-body',
      classNames: 'fade-in',
      toggleActions: 'add remove none none'
    }
  ]
});
```

### Toggle a Parent on Child Intersection

```js
it.add('.child-element', {
  toggleClass: [
    {
      targets: '.parent-container',
      classNames: 'child-visible'
    }
  ]
});
```

## How It Works

The four toggle actions correspond to the four intersection events:

1. **First action** -- trigger enters the intersection zone (`onEnter`)
2. **Second action** -- trigger leaves the intersection zone (`onLeave`)
3. **Third action** -- trigger re-enters from below (`onEnterBack`)
4. **Fourth action** -- trigger leaves back downward (`onLeaveBack`)

When `once: true` is set on the trigger, the class toggle happens only on the first enter event.

## Methods

### `kill()`

Cleans up the plugin, removing internal references. Called automatically when the IntersectionTrigger instance is killed.
